## CSS Porting Strategy

1. **Copy the entire `<style>` block** from the source verbatim into two files:
   - `app/styles/theme.css` — `:root`, `[data-theme="light"]`, `[data-theme="dark"]` blocks only.
   - `app/styles/landing.css` — everything else (reset, `.ascii-bg`, `.nav`, `.hero`, `.section`, `.building-*`, `.works-*`, `.contact`, `.footer`, `@keyframes`, `@media` blocks).

2. **Keep every class name and CSS variable name unchanged.** No renaming. No CSS Modules. No Tailwind.

3. **Import order in `root.tsx`:**
   ```ts
   import "~/styles/theme.css";
   import "~/styles/landing.css";
   ```
   Vite will bundle both into the emitted stylesheet.

4. The `[data-theme]` attribute lives on `<html>`. The root component renders `<html lang="en" data-theme="light">` for SSR; the no-FOUC inline script (see SSR Must-Handle #1) overrides it before paint; ThemeToggle updates it on click post-hydration.

5. **Do not** alter specificity, add CSS Modules hashes, or extract into component-scoped files — the source relies on global selectors like `[data-theme="dark"] .ascii-bg`.

## Five SSR Must-Handles

### 1 — Theme No-FOUC (Flash of Unstyled Content)

**Problem:** Server renders `data-theme="light"`. If the user's saved preference is `"dark"`, the page flashes light before JS hydrates.

**Solution:** Inject a blocking inline `<script>` as the first child of `<head>` in `root.tsx`. It must execute synchronously (no `async`, no `defer`).

```tsx
// root.tsx — inside <head>, before any stylesheet links
<script
  dangerouslySetInnerHTML={{
    __html: `(function(){
  var s=localStorage.getItem('theme');
  var p=window.matchMedia('(prefers-color-scheme: dark)').matches?'dark':'light';
  document.documentElement.setAttribute('data-theme', s||p);
})();`,
  }}
/>
```

- Server default: `<html data-theme="light">` (safe; overridden before paint if needed).
- `ThemeToggle` click handler: `document.documentElement.setAttribute('data-theme', next)` + `localStorage.setItem('theme', next)`. No React state for the attribute — it lives on the DOM element to match the source pattern exactly.
- **Pitfall:** Do not derive theme in a `useState` initializer — that runs after hydration and causes a mismatch. The DOM attribute is the single source of truth; React only observes it, never owns it.

### 2 — ASCII Black-Hole Background (Client-Only)

**Problem:** The animation uses `window.innerWidth`, `requestAnimationFrame`, `mousemove`, `textContent`, and reads `data-theme` at runtime. None of these exist on the server.

**Solution:**

- `AsciiBackground.tsx` renders `<div className="ascii-bg" id="ascii-bg" aria-hidden="true" />` on the server — an empty, styled div. No content.
- All animation logic lives inside a single `useEffect(() => { ... return () => { running = false; }; }, [])`. The effect is never called during SSR.
- Port the IIFE algorithm verbatim into the effect body. Preserve every constant (`BH_X`, `BH_Y`, `EH_RX`, `EH_RY`, `LENS_INNER`, `LENS_OUTER`, `GW_INTERVAL`, `charW`, `charH`, etc.) and every sub-function (`lensChar`, `ellipseDist`, `hash`, `easeOut`, `render`).
- **`prefers-reduced-motion`:** The source hides `.ascii-bg { display: none }` via `@media (prefers-reduced-motion: reduce)`. CSS handles this — no JS check needed. But additionally, guard the animation loop: check `window.matchMedia('(prefers-reduced-motion: reduce)').matches` at the top of the effect; if true, return early without starting `requestAnimationFrame`.
- **Theme awareness:** The `isDark()` helper inside the effect reads `document.documentElement.getAttribute('data-theme')` — identical to the source. No prop needed.
- **Hydration mismatch:** Because server renders empty div and client fills `textContent` only after mount, there is no hydration mismatch. React does not manage the `textContent` of this div.

### 3 — Montréal Clock (Client-Only)

**Problem:** Rendering the real time on the server causes a hydration mismatch (server time ≠ client time at hydration).

**Solution:**

```tsx
// Clock.tsx
export function Clock() {
  const [display, setDisplay] = useState("MTL --:--");

  useEffect(() => {
    function tick() {
      const opts: Intl.DateTimeFormatOptions = {
        timeZone: "America/Montreal",
        hour: "2-digit",
        minute: "2-digit",
        hour12: false,
      };
      setDisplay("MTL " + new Intl.DateTimeFormat("en-CA", opts).format(new Date()));
    }
    tick();
    const id = setInterval(tick, 30000);
    return () => clearInterval(id);
  }, []);

  return (
    <span className="nav-clock">
      <span className="dot-blink" />
      <span id="clock">{display}</span>
    </span>
  );
}
```

- Server renders `"MTL --:--"` — safe, no mismatch.
- `useEffect` fires after hydration, sets real time, then repeats every 30 s.
- **Pitfall:** Do not call `new Date()` during render or in `useState` initializer — that produces a mismatch between server and client render.

### 4 — Fonts via `<head>` Preconnect

Add in `root.tsx` `<head>`, before the stylesheet imports, exactly as in the source:

```tsx
<link rel="preconnect" href="https://fonts.googleapis.com" />
<link rel="preconnect" href="https://fonts.gstatic.com" crossOrigin="" />
<link
  href="https://fonts.googleapis.com/css2?family=Outfit:wght@400;500;600;700&family=Inter:opsz,wght@14..32,300..700&family=Geist+Mono:wght@400;500;600&display=swap"
  rel="stylesheet"
/>
```

Note: React requires `crossOrigin` (camelCase) not `crossorigin`.

### 5 — Meta / SEO via Route `meta` Export

```tsx
// app/routes/home.tsx
import type { MetaFunction } from "react-router";

export const meta: MetaFunction = () => [
  { title: "Humza Khan — Product Engineer" },
  {
    name: "description",
    content:
      "Product Engineer based in Montréal, working at the intersection of AI and finance.",
  },
];
```

React Router v7 merges route meta with root meta; ensure `root.tsx` does not duplicate title/description.

## Verification Procedure

### Setup

```bash
# Terminal 1 — React app
npm run dev          # wrangler dev; app on http://localhost:5173 (or as configured)

# Terminal 2 — original HTML (for visual reference)
npx serve /Users/humza/downloads   # or: python3 -m http.server 8888 --directory /Users/humza/downloads
# open: http://localhost:8888/humza-io-landing.html
```

### Screenshot Comparison (Three Breakpoints × Two Themes)

Use a headless tool (e.g. Playwright or puppeteer) or browser DevTools to capture screenshots at:

| Viewport | Theme |
|---|---|
| 1280 × 900 | light |
| 1280 × 900 | dark |
| 768 × 1024 | light |
| 768 × 1024 | dark |
| 390 × 844 | light |
| 390 × 844 | dark |

For each: open original HTML, screenshot → open React app, screenshot → overlay/diff (e.g. `pixelmatch`, `looks-same`, or `playwright expect(page).toHaveScreenshot()`). Accept threshold ≤ 1% pixel difference (font-rendering anti-aliasing noise is acceptable; layout/color/spacing differences are not).

Toggle theme by: (a) opening DevTools console and running `document.documentElement.setAttribute('data-theme','dark')` on the original HTML page, and (b) clicking the theme toggle button on the React app.

### SSR Smoke Test

```bash
# Verify no-FOUC script is present in raw HTML response
curl -s http://localhost:5173/ | grep -o 'localStorage.getItem.*data-theme'

# Verify server renders data-theme="light" by default
curl -s http://localhost:5173/ | grep 'data-theme="light"'

# Verify clock placeholder in SSR output
curl -s http://localhost:5173/ | grep 'MTL --:--'
```

### Reduced-Motion

Open browser DevTools → Rendering → "Emulate CSS media feature prefers-reduced-motion" → set to "reduce". Verify: ASCII background is hidden, dot-blink animation stops, all transitions are `none`.

## Stop Condition

Phase 1 is complete when:
1. All items in the Acceptance Criteria checklist pass.
2. Screenshot diff at all 6 viewport/theme combinations is ≤ 1%.
3. SSR smoke tests pass (no-FOUC script present, `data-theme="light"` in raw response, `"MTL --:--"` in raw response).
4. `wrangler deploy --dry-run` exits 0.
5. No TypeScript errors (`npx tsc --noEmit` exits 0).
